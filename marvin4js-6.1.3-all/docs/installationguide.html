Marvin for JavaScript - Installation Guide

Software requirement

Marvin for JavaScript is a web based editor. Please, upload it to a web server and refer to its URL in your browser.

Server side

  • Web server
  • JChem Webservices 2.0 is optional (if you need extended functionalities)

Client side

  • JavaScript enabled web browser, see browser compatibility

License requirement

Marvin for JavaScript requires Marvin4JS license. Consult with the sales team where you have to place the license file where your copy of Marvin for JavaScript package can access.

Please, take care that JChem Webservices 2.0 requires a separate license. Thus, you also setup a JChem Webservices for your site, you need JChem Webservices 2.0 license too. Consult with its manual how to setup its license.

Installation

Deploy Marvin4JS

Download the Marvin4JS zip package from the ChemAxon site.

Deploy Marvin4JS into an Apache Tomcat Container

JChem Webservice requires a servlet container like Apache Tomcat. If you would like to install both Marvin4JS and JChem Webservices, it worths to install both of them into the Tomcat Container. In this case, the web root can be an ideal place where you can deploy Marvin4JS:

  • Navigate into the webapps folder of Tomcat and enter its ROOT subdirectory.
  • Create a new folder in this directory (e.g. marvin4js).
  • Extract the Marvin4JS zip file into this new directory.

Deploy Marvin4JS under any web server

Extract the zip file into a new empty directory (e.g. marvin4js) and deploy it under your web server.
(Consult with the manual of the server or its administrator how to deploy new content there.)

Install web services for Marvin4JS

If you need extended functionalities, ChemAxon JChem Webservices 2.0 is required.

Marvin for JavaScript can communicate with other web service implementations that are compatible with the ChemAxon provided version. See the required web service specifications

  • Download ChemAxon JChem Webservices 2.0. We recommend that you use the same version than Marvin4JS.
    You can find the download link for the recommended webservice version on the Marvin4JS download site.
  • Install the required web services under the same domain as the web server hosts.
    The installation guide of JChem Webservices are available on the download page.
  • Marvin4JS searches web services on the following URL: /webservices2.
    If your web service copy is available on a different URL, please, modify settings in Marvin4JS.
    • Modify web service location in the webservices.js file, which you can find in the root of the Marvin4JS package.

Note: If web services and Marvin4JS are located on different domains, the browser can block the access of web services at runtime since the request may harm the same origin policy of the browser.

Try the deployed version

Open examples of Marvin4JS package from your web server in your browser to check whether the deployment was successful:

http://<web server url>/<marvin4js directory path>/examples/
